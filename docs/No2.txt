# ToGoGo Platform Development Session Summary

## Project Overview

ToGoGo is a PWA-based dropshipping/marketplace platform ("Where will you go?") with one-click automated store creation.

**Tech Stack:** React + Vite (frontend), Vercel serverless functions, Neon DB (PostgreSQL), Stripe payments, Google OAuth

**Live Site:** https://togogo.vercel.app | **GitHub:** https://github.com/comfybear71/togogo

**Current Status:** Early prototype, 156+ commits, multi-tenant with 5 supplier integrations (CJ Dropshipping, AliExpress, Printful, Printify, Gooten)


## Features Implemented This Session

### 1. Database Schema (10 Tables)
- `users` — Accounts with email/password (bcryptjs), Google OAuth, roles (admin/seller/buyer), wallet, trust score
- `user_products` — Products listed by sellers (supplier_cost, sale_price, category, platforms_listed)
- `user_orders` — Orders with commission tracking, supplier_cost, profit, tracking info
- `user_stores` — One-click stores with subdomains, themes, provision status, JSONB metadata
- `subscriptions` — Stripe subscriptions ($19.99 AUD/month)
- `platform_connections` — OAuth tokens for eBay, Etsy, Amazon, TikTok, WooCommerce
- `user_domains` — Custom domain registrations (Namecheap)
- `disputes` — Stripe chargebacks with evidence tracking
- `refunds` — Refund records
- `admin_settings` — Key-value config (commission rate, API keys, pricing rules)
- `categories` — 2-level hierarchy (parent + child categories with icons)
- `catalog_products` — Admin-managed global product catalog
- `supplier_product_cache` — Persistent product cache from supplier APIs (6-hour TTL)

**Schema Auto-Initialization:** ensureSchema() creates/updates all tables on first request


### 2. Authentication System
- JWT tokens with 30-day expiry, payload `{ id, email, role }`
- Password hashing with bcryptjs (12 rounds)
- Google OAuth full flow (redirect → callback → find/create user)
- Admin auth: JWT OR `x-setup-secret` header for initial setup
- Per-request auth via `authFetch` helper that auto-attaches Bearer token


### 3. Multi-Tenant Architecture
- Wildcard DNS `*.togogo.me` routes all subdomains to same deployment
- App.jsx detects subdomain from window.location.hostname
- StorefrontPage renders public shop for each subdomain (no auth required)
- Store data isolation via `user_stores.user_id` and `user_products.user_id`
- Store owner dashboard at `subdomain.togogo.me/dashboard` for authenticated owners


### 4. Store Owner Control Panel
- **Dashboard:** Sales stats, revenue charts, recent orders, top products
- **Products Page:** Manage listing visibility, add/edit/delete products
- **Orders Page:** Filter by status, view customer details, handle disputes
- **Settings Page:** Store name, theme selection, payment setup
- **Owner Toolbar:** Dark banner appears on storefront when owner is logged in, links to dashboard
- CORS headers configured for *.togogo.me origins


### 5. Admin Backend (/admin/ pages + /api/admin/* endpoints)
**Admin Navigation:** Dashboard, Users, Products, Orders, Stores, Marketing, Settings

**Pages:**
- Dashboard — Platform stats, user activity, revenue, charts, recent actions
- Users — User management, search, role/status updates, activity logs, pagination
- Products — Admin products with filters, category/supplier selection, commission breakdown
- Orders — Full order management, disputes, financials, customer tracking
- Stores — Store/domain management, health checks, provisioning status
- Marketing — Campaign analytics, referral tracking
- Settings — Commission rates, API key management, email templates, pricing rules

**Key Admin APIs:**
- `GET /api/admin/categories` — Fetch all categories
- `POST /api/admin/categories` — Create new category
- `PATCH /api/admin/categories/:id` — Update category
- `GET /api/admin/products` — Fetch all products (user + cache)
- `PATCH /api/admin/products` — Deactivate/activate/delete products
- `GET /api/admin/suppliers` — List all suppliers with product counts and status


### 6. Product Management & Supplier Integration

**Supplier Product Sourcing:**
- **CJ Dropshipping:** API key auth, keyword search, category browsing, 50-200 results per page
- **AliExpress:** App key/secret, feed-based browsing, multi-page fetching
- **Printful:** Full print-on-demand catalog (~400 products)
- **Printify:** Blueprint listing (~800 products)
- **Gooten:** Full recipe catalog (~300 products)

**All 5 suppliers normalized to common format:** id, title, description, image, supplier_cost, category

**Product Flow:**
1. Curated fallback products (60+ with Unsplash images) load instantly
2. Live API results merge in when supplier keys are configured
3. Products flow into supplier_product_cache table via `/api/dropship/cache-warm`
4. Cache persists across deployments with 6-hour TTL
5. Storefront serves products from cache → pricing engine applies markup → checkout

**Feed Endpoint Optimization:**
- POD suppliers (Printful/Printify/Gooten) return full catalog once instead of per-term searches
- CJ gets 3 bulk pages for popular discovery + keyword searches
- AliExpress fetches pages 1-2 from each feed
- Server-side caching: 5 minute in-memory cache, CDN cache 2 min with 10-min stale-while-revalidate
- 8-second timeout on all supplier API calls prevents page hangs


### 7. Storefront System
- Each store gets subdomain.togogo.me via Vercel wildcard domains
- **API:** `GET /api/storefront/store?subdomain=xxx` serves store info + products
- **Products on each storefront:** 1) Owner's custom products, 2) Supplier cache, 3) Catalog products, 4) Curated fallback
- Pagination, search, category filtering all supported
- CORS configured for *.togogo.me origins
- Gallery support: Auto-generates 5-image galleries using main image + 4 Unsplash variants if images array is empty


### 8. Pricing Engine
- Default 40% global markup (configurable)
- Per-supplier overrides in admin_settings
- Per-category overrides in admin_settings
- Applied automatically on storefront display and checkout
- Formula: `sale_price = supplier_cost + (supplier_cost × markup_rate)`

**Commission System:**
- Platform fee: 5% of sale_price (default, stored in admin_settings `platform_fee_percent`)
- Applied as `application_fee_amount` on Stripe destination charges
- Tracked per-order: `user_orders.commission` and `commission_rate`


### 9. Stripe Integration

**Subscription Flow:**
- Store creation requires $19.99 AUD/month recurring subscription
- `POST /api/subscriptions/checkout` creates Stripe checkout session
- Auto-creates Stripe Product + Price on first use, caches price_id in admin_settings
- Webhook handles: checkout.session.completed, subscription lifecycle, invoice events
- Subscription status: active → past_due → cancelled → expired
- User role auto-upgraded to 'subscriber' on purchase, downgraded on cancellation

**Webhook Handler (/api/webhooks/stripe):**
- Signature verification with STRIPE_WEBHOOK_SECRET
- Handles 13+ events: checkout.session.completed, subscription.created/updated/deleted, invoice.payment_succeeded/failed, charge.dispute.created, charge.refunded
- Webhook security: Raw body required (config.api.bodyParser = false)

**Stripe Connect (Partially Implemented):**
- `POST /api/connect/onboard` — Creates Custom connected accounts for store owners
- `POST /api/connect/account-session` — Embedded onboarding components
- `GET /api/connect/status` — Check account status, balance, earnings
- `GET /api/connect/dashboard` — Account details, payouts, recent charges
- Storefront checkout uses destination charges: `payment_intent_data.transfer_data.destination` = store's stripe_connect_id
- Webhook syncs account.updated events

## Architecture Summary

**No Express Server:** Each file in `/api/` is a separate Vercel serverless function

**Data Flow:**
1. Supplier APIs (CJ, AliExpress, Printful, Printify, Gooten)
2. → Curated products (fallback with Unsplash images)
3. → supplier_product_cache table (via cron or cache-warm endpoint)
4. → Storefront API serves to stores
5. → Pricing engine applies markup
6. → Checkout creates Stripe session
7. → Payment flows: customer → platform (takes commission) → store owner (via Connect)

**Webhook-Driven:** Store activation triggered by Stripe webhook after payment received

**Database Auto-Migration:** Schema creates tables with CREATE TABLE IF NOT EXISTS, no manual migrations needed

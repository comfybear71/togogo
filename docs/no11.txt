Project Overview
ToGoGo is a dropshipping & marketplace PWA platform. Store owners sign up, pay $19.99 AUD/month, get a subdomain.togogo.me storefront auto-provisioned. Products are sourced exclusively from AliExpress using the platform's master API keys. Customers buy from storefronts, platform takes 30% of profit (sale_price - supplier_cost) as commission, store owners get the rest via Stripe Connect. Orders are auto-created on AliExpress with customer shipping address — admin pays in bulk.
Live site: https://togogo.me
Storefronts: https://stu.togogo.me, https://jum.togogo.me, https://stuie.togogo.me, https://annies-shop.togogo.me

The Extra Setting/API Part You're Likely Missing
Even with Auto-pay authorized in the Business portal, the API call that creates or places the order on AliExpress often needs an explicit payment initiation step (or specific parameter) to trigger the auto-charge. Simply creating the order via API doesn't always auto-pay — many integrations leave it in "Awaiting Payment" unless the payment request is sent correctly.
From AliExpress's own description:
Auto-pay in the Business section is designed specifically "to initiate payment request via our dropshipping API".
You bind/authorize a payment method there (ideally the PayPal you have activated, as it tends to be more reliable for auto than raw credit cards).
Then, when your system/tool pushes the order, the API call should include (or follow up with) a payment request that leverages that authorization.
Common missing pieces in the API flow:
After creating/placing the order (e.g., via order creation or fulfillment endpoint), you (or your tool) need to call a payment-related API to request the charge against the authorized Auto-pay method.
Some dropshipping tools handle this internally if configured, but if you're using a custom script or partial integration, the payment request step can be separate.
For credit cards specifically: They can trigger extra verification (bank 3DS, CVV re-entry, or fraud checks), so orders often still require manual intervention even with Auto-pay enabled. PayPal is usually smoother for true hands-off auto-debit.

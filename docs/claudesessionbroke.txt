How do my clients log into the admin panel?


Based on the codebase, there are two ways to access the admin panel:
Database role — Set a user's role column to 'admin' in the users table. They then log in normally (email/password or Google OAuth) and the frontend AdminRoute component checks user.role === 'admin' to grant access to /admin/* routes.
Setup secret header — For API-only access (e.g., initial bootstrapping before any admin user exists), send the x-setup-secret HTTP header with the value of your JWT_SECRET environment variable. This bypasses the JWT/role check on admin API endpoints.
There is no dedicated admin login page — admins use the same /auth sign-in page as everyone else. The difference is purely the role field in the database.
To make someone an admin, you'd run something like:
UPDATE users SET role = 'admin' WHERE email = 'their@email.com';

Or use the seed script to create a test user and then manually promote them.

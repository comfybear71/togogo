ToGoGo Platform — Comprehensive Context & Next Steps
Repository: togogo (GitHub: comfybear71/togogo)

Domain: <togogo.me> (Australian dropshipping/e-commerce platform)
WHAT THIS PROJECT IS
ToGoGo is a dropshipping platform where:
1. Clients (store owners) sign up, pay $19.99/mo AUD subscription, get a <subdomain.togogo.me> storefront auto-provisioned
2. Customers browse products on those storefronts and buy via Stripe checkout
3. Platform admin manages everything via /admin dashboard
4. Products are sourced from AliExpress and CJ Dropshipping APIs using the platform’s master API keys (users never need their own supplier accounts)

The orderRefFromUrl = useMemo(...) was on line 1297 — AFTER the if (loading) return ... and if (!storeData) return ... early returns. This violates React's rules of hooks:
Render 1: loading=true → early return at loading check → N hooks executed
Render 2: loading=false, data loaded → passes early returns → hits the extra useMemo → N+1 hooks → CRASH
React requires the exact same hooks to run in the exact same order on every render. A hook after an early return only runs sometimes.
Fix: Moved orderRefFromUrl useMemo up with all the other hooks, before any conditional returns.
Also fixed: The service worker was aggressively caching ALL JS/CSS bundles (precache 81 entries), so your browser kept serving old buggy bundles even after deploys. Now JS/CSS uses NetworkFirst strategy so deploys take effect immediately.
For your browser right now: You may need to clear the old service worker cache one more time:
Chrome DevTools → Application → Storage → "Clear site data" (check all boxes)
Then reload jum.togogo.me
After this deploy, the storefront should work reliably.
